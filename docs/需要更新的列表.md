# 更新日志
## 版本更新描述
自从v0.1.6版本之后，每一次更新记录需要在更新日志中记录，并且按照工作模式逐步排查问题，更新按照 修改/新增/删除/重构 四个模式进行。
其中修改应对小内容修改，重写指的是对一个大模块的重构，内容量较多，新增和删除顾名思义。

## commit 标准
每一次代码提交commit
如：
- [fix] $(fileName): add a function to fix bug.
- [类型] $()
这样的模式，按照 [描述类型] $(影响范围) 具体修改的内容 这样表达.
